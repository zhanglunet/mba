# Dimension 6 — Channel / 渠道
# Kimi / 月之暗面 Brand Audit · Phase 2F
# Audit Date: 2026-06-26

## Core Finding

Kimi 的渠道是典型的"直接触达用户"D2C 模式：kimi.ai 网站 + Kimi App + API/开发者平台。这种渠道在短期有效（用户自发口碑传播），但缺乏任何"平台寄生型"分发渠道——没有嵌入微信、抖音、微博的流量入口，而这些是字节和阿里的核心武器。

## 渠道矩阵

| 渠道 | 用户数 | 品牌价值 | 可规模化 |
|------|--------|----------|----------|
| kimi.ai（Web）| 高（网页端是主力）| 高 | 中 |
| Kimi App（iOS/Android）| 中 | 高 | 中 |
| Kimi API（开发者）| 中 | 高（技术圈）| 高 |
| 第三方嵌入/集成 | 低 | 中 | 有待开发 |
| 微信/抖音生态 | 几乎无 | — | 需要合作 |

## 渠道劣势：分发鸿沟

| 渠道 | Kimi | 字节 Doubao |
|------|------|-------------|
| App Store 自然流量 | 有限 | 强（字节 App 矩阵导流）|
| 微信/微信生态 | 无直接入口 | 部分合作 |
| 抖音/视频号 | 无 | 深度绑定 |
| 企业微信/钉钉 | 有限集成 | 字节飞书生态 |

## 开发者渠道（相对优势）

- Kimi API 在中国开发者社区有较高的品牌认知
- 文档质量、API 稳定性、中文理解质量均有口碑
- 相对于字节 Doubao API，Kimi API 更受"技术纯度"需求者青睐
- 这个渠道对长期品牌建设有价值，但规模有限

## Brand Signal Score

**6 / 10** — 渠道结构对于独立 AI 公司是合理起点，D2C + API 是可以建立忠诚用户群的路径；但没有任何超级 App 生态嵌入，意味着增长天花板明显低于字节/阿里系产品。
