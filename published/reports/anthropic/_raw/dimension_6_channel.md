# Dim 6 — Channel Architecture / 渠道架构

## Summary

Anthropic 建立了 AI 行业最宽的渠道覆盖：唯一同时在 AWS Bedrock、Google Cloud Vertex AI、Microsoft Azure Foundry 三大云平台上线的前沿模型，触达全球企业的既有云合约，零新增采购摩擦。叠加 Claude Code 的开发者生态、Claude.ai 的消费端、Claude Partner Network 的 ISV 层，形成了四层并行的渠道架构。渠道广度已超过 OpenAI（主要绑定 Azure）和 Google（主要自营 Vertex），是 2026 年企业采购首选的核心结构性原因。

## Key Evidence

**三云覆盖（唯一性）**
- AWS Bedrock + Google Cloud Vertex AI + Microsoft Azure Foundry 全覆盖
- 企业无需新采购关系，用既有云合同即可访问 Claude
- 来源：[CloudComputing News](https://www.cloudcomputing-news.net/news/claude-platform-on-aws-anthropic-partnership/)

**Amazon 战略级绑定**
- Amazon 已投资 $8B，追加承诺至 $5B 新增（最高 $20B 上限）
- AWS 十年技术支出承诺超 $100B，锁定 5GW 算力
- 100,000+ 企业客户通过 Amazon Bedrock 使用 Claude
- 来源：[Anthropic/Amazon](https://www.anthropic.com/news/anthropic-amazon-compute)

**合规渠道**
- Claude in Bedrock 获 FedRAMP High 和 DoD IL4/IL5 认证
- 直接打开美国联邦政府和国防部市场——这是竞争对手尚未触达的渠道
- 来源：[Anthropic](https://www.anthropic.com/news/claude-in-amazon-bedrock-fedramp-high)

**Claude Partner Network**
- 2026 年 $100M 生态系统投资计划
- ISV / 系统集成商 / 垂直行业合作伙伴层建设中
- 来源：[Pooya Blog](https://pooya.blog/blog/anthropic-claude-partner-network-100m-2026/)

**Claude Code 开发者渠道**
- Claude Code 已成独立产品，ARR $2.5B，在 IDE/CLI 层直接接触核心开发者
- 开发者一旦建立工作流依赖，转换成本极高，是最黏性的渠道

**两种 AWS 访问模式**
- Claude Platform on AWS：原生体验 + beta 功能优先访问
- Claude on Bedrock：数据留在 AWS 内，Anthropic 不接触客户数据——满足最严格数据合规需求

## Brand Signal

**渠道广度评分：9/10**

三云全覆盖 + FedRAMP 合规 + $100M Partner Network 是系统性渠道建设，而非临时合作。唯一弱点是消费端（Claude.ai）订阅数据未公开，C 端渠道强度不明。

## Gaps / Uncertainties

- Google Cloud 合作深度和 Vertex AI 上的客户数量未公开
- Claude.ai 消费端订阅人数、ARPU 未披露
- $100B AWS 承诺中有多少是 committed spend vs aspirational target，尚待 IPO 招股书澄清
