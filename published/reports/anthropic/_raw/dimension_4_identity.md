# Dim 4 — Identity System / 身份系统

## Summary

Anthropic 采用双品牌架构：Anthropic 是 B2B/安全研究的公司品牌，Claude 是面向用户和开发者的产品人格品牌。"Claude"的命名策略是业内少有的为 AI 赋予独立人名（而非产品线编号），配合文学诗体命名体系（Haiku/Sonnet/Opus/Fable），构建了一套有审美感的产品身份。但双品牌架构带来认知分裂：大众知道 Claude 但不知道 Anthropic，企业买家认识 Anthropic 但对 Claude 人格感知淡薄。

## Key Evidence

- **人名策略**："Claude" 是一个法语/英语男性名，让 AI 具有明确人格锚点。与 GPT（技术描述词）和 Gemini（星座/品牌词）相比，人名策略更易产生拟人化情感连接 — 来源：[Claude.com](https://claude.com/resources/tutorials/choosing-the-right-claude-model)
- **文学诗体命名**：Haiku（俳句 - 快速/轻量）/ Sonnet（十四行诗 - 日常/平衡）/ Opus（作品 - 专业/深度）/ Fable（寓言 - 最高阶），命名传递能力层级且自带美学调性，与智能手机产品线（Pro/Max）逻辑类似但更有文化厚度 — 来源：[Second Talent](https://www.secondtalent.com/resources/every-claude-ai-model-explained-compared/)
- **模型命名一致性**：4.x 家族（Claude Haiku 4.5 / Sonnet 4.6 / Opus 4.7/4.8）保持命名体系连贯，产品线清晰，对比 OpenAI 的 GPT-4/o1/o3/4o 命名混乱有显著优势 — 来源：[Anthropic Docs](https://platform.claude.com/docs/en/about-claude/models/overview)
- **Claude.ai vs API 分裂**：消费端 Claude.ai 强调人格和对话体验；开发者端 API/docs 强调能力和工程特性；两套语言风格不同，人格一致性弱于 ChatGPT 的单一入口策略
- **Anthropic 公司品牌淡薄于 Claude**：大众用户感知"Claude"强于"Anthropic"，类似 Google 强于 Alphabet——这是产品品牌成功但公司品牌未跟上的典型案例

## Brand Signal

**身份系统评分：7/10**

命名策略和产品线架构是业内最美学、最连贯的，这是真正的品牌资产。但 Anthropic/Claude 双品牌的认知分裂是实质性弱点——在 IPO 路演中，公司品牌和产品品牌需要融合，目前存在叙事断层。

## Gaps / Uncertainties

- Claude 的具体"人格定义"（价值观/性格/边界）没有像 ChatGPT 的 system card 那样的公开人格文档
- Fable 5 是否会延续诗体命名体系或标志新阶段，尚不明确
- 双品牌架构在 IPO 后的整合策略未公开
