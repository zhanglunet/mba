# Dimension 2 — Category / 品类定义
# 奇安信 Brand Audit · Phase 2F
# Audit Date: 2026-06-26

## Core Finding

奇安信是中国"政务级网络安全"品类的实际定义者与市场领导者——这是一个真实存在的巨大类别，但也是一个高度中国特殊化的品类，天然限制了国际品牌溢出。

## 品类定义层次

### 层次 A：核心品类 — 政务/政府安全
- 中国 80% 以上中央政府机构是奇安信客户（据公司年报及行业报告）
- "政务安全"不是通用网络安全市场的子集，而是中国特有的监管驱动市场
- 核心竞争力：符合国家等保（等级保护）2.0 / 分级保护 / 关基保护要求
- 奇安信在这个细分品类中基本没有旗鼓相当的国内竞争对手

### 层次 B：扩展品类 — 下一代安全操作系统（2022+）
- XDR（扩展检测与响应）+ Zero Trust（零信任）+ AISOC（AI 安全运营）
- 试图从"政务安全供应商"升级为"安全操作系统平台"
- 对标：Palo Alto Networks Cortex、Microsoft Sentinel、CrowdStrike Falcon

### 层次 C：AI+安全融合品类（2023+）
- 发布"奇安信安全大模型"，将 LLM 技术融入安全运营
- 目标：AISOC 将安全运营中心自动化，减少人工成本
- 国内首批将 LLM 落地到安全运营场景的企业之一

## 品类竞争格局

| 品类 | 奇安信位置 | 主要竞争者 |
|------|------------|------------|
| 中国政务安全 | 第一 | 360安全、深信服、华为云安全 |
| 中国企业 XDR | 第一梯队 | 深信服、绿盟、安恒 |
| 全球企业安全 | 不存在 | CrowdStrike、Palo Alto Networks、Microsoft |
| AI安全/AISOC | 中国第一梯队 | 安恒信息（本土），全球: CrowdStrike Charlotte AI |

## 品类护城河分析

**强护城河**：政务安全资质（保密资质、等保测评权、国家指定安全厂商地位）——这些是监管壁垒，竞争对手极难复制。

**弱护城河**：技术层面，XDR/Zero Trust 的核心逻辑全球标准化，国际厂商理论上可以用"不受安全禁令限制的技术"进入中国市场（但实际存在数据主权限制）。

## Brand Signal Score

**7 / 10** — 政务安全品类定义扎实，市场地位可核查，但品类天花板由中国政府政策决定，不是技术或品牌的选择结果；升级为"安全操作系统平台"的叙事仍在验证阶段。
