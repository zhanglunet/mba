MBA 是 AI 时代的
品牌影响力审计协议。

不是写一份品牌报告 —— 而是把一家公司在 Agent 世界里的影响力， 拆成 7 个调研维度 × 5 个打分镜头， 由 Lead 编排、子 agent 并行调研、N 位人物评委独立打分， 输出 Markdown + 自包含 HTML 的版本化报告。

01

The Reframe.

品牌正在从"人类心智资产"变成"智能体世界里可识别、可比较、可调用的信号资产"。

升维

过去品牌影响 人。
未来还要影响 Agent。

人类看到的

故事、审美、身份、文化符号。slogan / logo / PR / 社媒声量。

Agent 看到的

稳定叙事、清晰品类、高质量来源反复引用、竞品对比中的独特位置、 真实用户与真实增长。结构化信号。

投资 Agent 筛公司 · 采购 Agent 推荐供应商 · 招聘 Agent 理解雇主品牌 · 消费 Agent 比较产品 · 战略 Agent 持续追踪格局

— Chapter

02

The Problem.

财务有审计、产品有数据分析、销售有 CRM、舆情有 monitoring。品牌影响力没有审计协议。

三个顽疾

传统报告的三条断裂。

01

single-thread

单线程单视角。一个分析师写一份报告，容易陷入自家叙事或个人偏好。

02

non-replayable

不可复盘。报告一次性，半年后再看，不知道哪些结论已经过时。

03

drifting metric

口径漂移。每个分析师有自己的尺度，跨品牌、跨时间不可比。

03

The Pipeline.

Lead 编排 · 子 agent 并行调研 · 人物评委独立打分 · 版本化报告滚动覆盖。

五阶段流水线

六个 Phase。一条流水线。

PHASE 0
Router

Lead 检查 reports/<brand>/report.md 是否存在 → FRESH 或 EVOLUTION 模式。

PHASE 1
Discovery

Lead 起草 PRD：定位 / 7 维度 / 评委 panel / 是否走云浏览器。

Gate 1 · 用户确认
PHASE 2
Parallel Search

N 个 sub-agent 并行调研，每维度 1 个 + 可选 1 个 Wuying 云浏览器 leg。

PHASE 3
Synthesis

Lead 读所有 _raw/ → synthesis.md：杠杆地图、脆弱边缘、跨维度矛盾。

PHASE 4
N-Judge Panel

N 位评委各自 LOAD perspective skill，5 镜头 1-10 分独立打分。

PHASE 5
Lead Merge

合并 synthesis + reviews → report.md / report.html / versions/v{n}。

EVOLUTION 模式跳过 PRD 起草，改为 diff plan：只重跑变了的维度 · 版本号 +1

演化模式

每个品牌一条时间序列。

FRESH 模式

没有报告时 —— Lead 走完五阶段全流程，落 v1 不可变快照。

reports/openclaw/
├── report.md          ← 滚动覆盖
├── report.html        ← 自包含 HTML
├── panel.yaml         ← 锁定哪套评委
├── versions/
│   └── v1_2026-06-12.{md,html}
├── _raw/                ← 过程文件
└── reviews/             ← 评委独立打分卡

EVOLUTION 模式

已有报告时 —— Lead 跑 diff plan：自动识别哪些维度可能变了， 只对变了的维度 delta 调研、让评委只对变了的维度重打分， 写入 v{n+1}。 

versions/
├── v1_2026-04-10.{md,html}  ← 初版
├── v2_2026-05-09.{md,html}  ← delta on dim 3,7
└── v3_2026-06-12.{md,html}  ← delta on dim 1,4

真正有价值的不是"今天几分"，而是"过去 90 / 180 天发生了什么结构性变化"

04

The Output.

一份 Markdown 给 LLM · 一份自包含 HTML 给人看 · 一份 _raw/ 给下一次 EVOLUTION。

报告解剖学

HTML 报告的 10 个区块。

01 · HERO

品牌名 + 版本 badge

mode 标签 + 一句 TL;DR；不堆图。

02 · SCORE RADAR

5 镜头雷达图

Chart.js radar，每评委一彩色多边形，共识区一眼可见。

03 · DISSENT HEATMAP

5 × 5 异议热力图 + σ 列

低 σ 行 = 共识、高 σ = 撕扯，扫一眼最高价值的 surface。

04 · INFLUENCE FLOW

影响力构造图

Mermaid flowchart：维度做源 → 放大器 → 可观察表面，边标 leverage 假设。

05 · POSITIONING QUADRANT

品牌定位象限

Lead 选两条轴，品牌 + 3-5 竞品定位，Mermaid quadrantChart。

06 · JUDGE CARDS

N 张评委卡

portrait + verdict + 金句 + 5 镜头分数行 + 卡片边框按总分四分位染色。

07 · BRAND MINDMAP

品牌精髓脑图

品牌为根 · 7 维度为枝 · 每枝顶 2-3 finding。

08 · ACTION RECS

90 天行动建议

编号 + leverage badge（high/med/low），按杠杆排序。

系统字体 · 离线可读 · prefers-color-scheme 自动深色 · @media print 友好 · 最大宽度 960px 抗"AI 无脑全宽"

最高价值的扫一眼界面

异议热力图

06

The Audience.

不卖给泛市场部。卖给真正关心"品牌如何转化为战略资源"的人。

三类用户

谁在用，用来回答什么。

PRIMARY · INVESTORAI native 早期投资人 / FA
一周看 5-10 个 pitch，品牌叙事真假难辨。增长来自真实势能，还是渠道红利？
25 分钟出一份带 5 评委独立打分的 Brand Influence DD 报告。
PRIMARY · FOUNDER创始人 / CEO Office
融资 deck、官网、产品 demo、销售材料、媒体采访 —— 故事太多，互相矛盾。
每月一次 EVOLUTION，把品牌当镜子，盯 90 天结构性变化。
SECONDARY · CREATOR内容创作者 / 研究员
写品牌深度文之前需要事实参考；研究框架需要可复用的"评估组件"。
可复用的品牌评估组件，嵌入研究工作台或 agent 框架。
商业化路径

三层 —— 从单次审计到Skills 平台。

01

02

L2 · Monitor
持续监控订阅

对基金监控 portfolio / watchlist。对上市公司监控自己与核心竞品。MBA 从一次性报告 → 品牌影响力时间序列。

L3 · Platform
Skills 平台化

Brand Audit · Category Mapping · Narrative Diff · Competitor Benchmark · IPO Narrative Check · Founder Story Audit —— 拆成可调用 skills 接入企业知识库 / 投研系统。
