项目周报

Weekly Report · 面向非技术读者，聚焦业务价值

本周（截至 2026-07-19） 里程碑 · v0.5.0 关系宇宙 + v0.6.0 舆情自动化闭环 · 审计品牌 15→24 · 创始人/晚餐/产业三维上线 · 舆情从人工核验升级到 AI 自动分类

一、本周一句话总结

本周连发两个大版本：先把 MBA 从"给一个个品牌单独打分"扩成一张互相关联的关系网络——每个品牌接上创始人、创始人之间能摆"晚餐"推演合作、品牌按产业归类，并一口气把九家全球巨头纳入监控（审计品牌 15→24）；再把舆情监控从"人工一条条核验"升级成每天自动跑的流水线——AI 预先分类、自动整理入库、人只需审最后一版改动点头。全程守住那条底线：AI 只做分类判断、从不改动审计分数，合并入库永远是人在把关。

二、解决了哪些问题 / 满足了哪些需求

1"只知道品牌、不知道品牌背后的人" → 创始人维度上线，每个品牌接上创始人

过去报告只评品牌，但品牌的叙事往往和创始人深度绑定。本周从零建成创始人维度：为全部 24 个品牌梳理创始人履历时间线（每条都带公开来源出处），并从人物视角刻画"创始人↔品牌"的关系。网站上每位创始人有独立可点开的档案页。反捏造铁律照旧：关系描述强制以"分析:"开头（机器门禁拦截），承认这是推演而非事实断言。

2"品牌之间能不能碰出合作" → 创始人晚餐:把两位创始人放一桌推演合作

新增创始人晚餐玩法：挑两位创始人放同一张桌子，按五个观察镜头假想推演他们的品牌能怎么合作。本周上了 3 场（田溯宁×唐杰、梁文锋×黄仁勋「芯片与模型」、任正非×马斯克）。首页嵌入"晚餐 · 亮点"轮换块，网站还做了组合器：想看的组合有现成的就"开饭"、没有的一键"点单"（自动预填下单表单）。明确标注：发言是 AI 演绎公开立场、合作是假想、绝不谎称真实合作。

3"品牌一大堆、没法按行业看" → 产业维度 + 首页筛选

品牌数量涨上来后，混在一起不好找。本周加产业维度：把品牌分成六大类（人工智能 / 消费 / 硬科技·航天 / 智能制造·硬件 / 企业服务·安全 / 教育），首页加产业筛选条 + 每张卡片带产业标签，点一下只看某一产业。归类进机器一致性门禁，保证每个品牌都有合法产业、不漏不错。

4"监控名单太窄、缺全球头部" → 九家全球巨头入库，审计品牌达 24

本周新增 9 家全球审计品牌：DeepSeek、NVIDIA、Apple、Google、Microsoft、Amazon、Huawei、Tesla、Palantir——每家含完整审计报告（数字自洽的评分矩阵 + 内联图表）+ 创始人档案。其中 4 家首次真实触发"利益回避"机制（评委本人是该品牌创始人/董事就自动剔除、不给自己打分）：Microsoft（霍夫曼曾任微软董事）、Huawei（任正非）、Tesla（马斯克）、Palantir（Thiel）。另有 8 个新品牌先上舆情驾驶舱冷启动（27 条可溯源事件），审计排期在后。当前审计史最高分是 Apple 8.84。

5"舆情大事还得靠人一条条核验、太累" → 前台打勾 + AI 自动分类,人只审最后一版

这是本周下半程的重头。舆情监控原来的流程是：系统每天发现候选新闻 → 人打开一堆候选、逐条判断、手工粘贴入库，很费事。本周分两步彻底改造：

  • 第一步·前台 Triage 页：候选新闻做成卡片，访客直接打勾采纳 / 打叉丢弃、就地改判断字段，一键提交入库（取代"复制粘贴 + 读代码改动"）；候选标题全部中文化。
  • 第二步·AI 全自动分类：每天自动发现候选（过滤掉股票行情页这类噪音）→ AI 预先分类（判断维度/严重度/方向/影响镜头）→ 自动整理入库、开一个"建议入库"的待审提交 → 人只需看最后一版改动、点头合并。没配 AI 时优雅兜底：候选照样送到前台供人工筛。

6"品牌变了,分数还是旧的" → 4 次真实重审,把已发生的大事消化进评分

监控发现大事不是终点，重审才是。本周用演化追踪模式做了 4 次品牌重审（v1→v2），把已入库的重大信号消化进评分：

  • Kimi：K3 模型发布把"品牌>业务信号"翻盘，6.85→7.50；
  • DeepSeek：自研芯片补上最弱的一环 + 融资/IPO 消息，8.00→8.28；
  • Microsoft：对 OpenAI 的依赖疑虑 + 版权诉讼坐实，8.65→8.55；
  • NVIDIA：对华出口管制风险坐实，8.88→8.80（因此退居第二，Apple 登顶）。

每次都留下版本快照与对比，方向和数字都在报告里可回溯。

7"两版都发出去了,网站/文档得跟上" → 发版 + 截图 + 文档保鲜

本周正式发布 v0.5.0 与 v0.6.0 两个版本（各自切出带更新日志的正式发布），并更新 README 首页截图到最新首页。文档保鲜：把顶部快照里"NVIDIA 现居最高分"这类被重审改掉的旧数字，逐一更正为实况（Apple 8.84 登顶）——数字必须对得上仓库真实状态。

四、质量保障

  • 反捏造机器门禁全程在岗：引用逐字命中语料（本周 200+ 条引用校验）、创始人关系强制"分析:"前缀、舆情标题/日期/链接逐字取自来源；判断字段明确标注为"模型判定、不假装客观"。
  • 自动分类踩坑如实记录并绕过：AI 网关先后遇到限流（改走另一端点 + 退避重试）、密钥端点配错（定位到配置项）、机器人提交的待审被平台默认卡住（改成开单前就本地整理好、绕开卡点）——每个坑都定位到根因再修，不含糊过去。
  • 一致性守卫抓出真实滞后：发版时机器门禁抓到"首页信号条没跟上新入库事件"的滞后（此前只改数据的提交不触发该检查而漏到现在），当场重生成补齐、复绿。
  • 数字一律从仓库实况核对：本周报所有数字（24 品牌 / 24 创始人 / 3 晚餐 / 4 重审 / 各品牌新旧分数）均按仓库当前文件逐一核实，不凭印象。

五、方法论沉淀

  • 自动化不越反捏造的线：把"人一条条判断"换成"AI 预分类"，但严格切分——标题/日期/链接是逐字搬运的客观事实，维度/严重度/方向是明确标注的模型判断，审计分数从不自动变，合并入库永远是人在把关。自动化提效率，人守最后一道闸。
  • "先认账、再调参"延续：网关抖动、配置错、平台卡点——本周每个自动化坑都是先如实定位根因、再结构性绕过，而不是把失败藏起来重试到过为止。
  • 单一真源 + 机器一致性守卫：产业、创始人、晚餐、版本号都收敛到唯一数据源，页面由脚本生成；一致性门禁负责抓"数据改了、页面/文档没跟上"的漂移——本周它就抓出了首页信号条的真实滞后。
  • 关系图谱的诚实边界：创始人晚餐这类"推演"内容，用强制的"分析:"前缀、"诚实盒"（并列合作张力）、"AI 演绎公开立场"标注，把"假想推演"和"事实断言"在产品层面清清楚楚分开。

六、当前状态 & 下周展望

当前：v0.6.0 已发布上线；舆情自动化流水线跑通（每日自动发现 → AI 分类 → 自动入库 PR → 人审 diff），审计覆盖 24 品牌 + 24 创始人 + 3 场创始人晚餐 + 6 大产业，AI 网关配好后可全自动。

下周可推进（部分需外部资源配合）：

  • 让每日自动分类 PR 稳定产出、观察 AI 网关可用性（本周遇到过瞬时不可达，已有兜底）；
  • 8 个已建驾驶舱的品牌补完整审计报告；
  • 更多创始人晚餐组合（组合器"点单"已可用，按需上桌）；
  • 随舆情信号累积到阈值，触发下一轮品牌重审。
